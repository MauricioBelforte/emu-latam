### 🚨 NUEVA OBSERVACIÓN DEL USUARIO

El usuario nota que el doble-input parece estar **vinculado al timing de soltar la tecla**. Sensación: cuando se presiona una flecha, el host ve el movimiento una vez. Pero **al soltar la tecla**, es como si el guest presionara otra vez. Como si el evento "key release" en el guest se interpretara como un "key press" adicional en el host.

Esto se alinea más con un **problema de netplay state synchronization** que con conflicto de polling o drivers.

### Probables causas restantes

1. **El guest envía su estado local** (incluyendo inputs) al host. El host lo recibe y lo aplica. Pero RetroArch netplay tiene un mecanismo donde el **host re-ejecuta frames** del guest para sincronizar. Si el host aplica el input del guest **y además** aplica su propio polling de red, podría generar el doble-input.
2. **Problema conocido de netplay de RetroArch con WiFi**: jitter variable causa que el host reciba paquetes desordenados, y el sistema de rollback/sync de RA re-procesa frames con los inputs del guest ya aplicados.
3. **El `check_frames = 3` combinado con WiFi** hace que el host verifique el estado del guest cada 3 frames, y si hay diferencia, fuerza una re-sync que re-aplica inputs.
