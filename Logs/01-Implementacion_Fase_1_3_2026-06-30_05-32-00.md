## Descripción de los Cambios

Se implementaron las mejoras de la Fase 1.3 enfocadas en la experiencia del usuario y en mitigar el lag del Netplay en RetroArch.

### 1. Inyección de Configuración Anti-Lag (Run-Ahead)
- Se creó el archivo `retroarch/netplay_optimized.cfg` con parámetros avanzados de rendimiento y reducción de frames de latencia (Run-Ahead, GPU hard sync, frame delay) optimizados para el core FBNeo jugando KOF '98.
- Se modificó `client/src/main/index.ts` para verificar la existencia del archivo e inyectarlo dinámicamente usando la bandera `--appendconfig` de RetroArch.

### 2. Prevención de Login Rápido (Nakama Health Check)
- Se agregó el handler IPC `check-nakama-health` en Electron que consulta asíncronamente el estado de Nakama local.
- Se implementó un hook `useEffect` de consulta periódica en `client/src/App.tsx`.
- Se inhabilitó el botón "INSERT COIN" al inicio con estilos CSS de styled-components específicos para deshabilitado (`:disabled`), mostrando una animación CSS de pulsación suave mientras el servidor termina de arrancar.

### 3. Join Directo (IP Express)
- Se automatizó el almacenamiento local en `localStorage` de la dirección IP de Relay cuando un invitado presiona "2. JOIN GAME", eliminando el paso obligatorio de presionar "GUARDAR CONFIGURACIÓN".
